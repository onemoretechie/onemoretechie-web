## What's covered under Security

The security work that Principal Cloud Architects actually own — access, secrets, compliance, and the hard trade-offs between velocity and risk.

### Core areas

- **IAM at scale** — SSO, SCPs, permission boundaries, cross-account role assumption
- **Zero-trust** — network segmentation, workload identity, service mesh security
- **Secrets management** — AWS Secrets Manager, Vault, rotation policies
- **Compliance** — SOC 2, PCI, HIPAA controls that actually map to AWS services
- **Threat modelling** — STRIDE, attack trees, applied to cloud-native systems
- **Detection & response** — GuardDuty, Security Hub, CloudTrail investigations

### The framing

Security work that visibly slows delivery hasn't landed properly. The goal is guardrails engineers barely notice — not gates they route around.
